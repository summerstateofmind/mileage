# Mileage V0.1 MVP Implementation Plan

> **For agentic workers:** Execute task-by-task. Mark each checkbox as you go. After each task, run the verification step before moving on.

**Goal:** Build the smallest version of Mileage where `mileage sync && mileage show` reads the author's actual Claude Code JSONL + a local git repo and prints a real YPT number.

**Architecture:** Approach A pipeline — Ingest (Claude Code JSONL + git) → Compute (YPT formula + provenance) → Render (terminal text). SQLite at `~/.mileage/metrics.db`. Synchronous, single-process CLI.

**Tech Stack:** TypeScript 5.6+, Node 18+, `commander` for CLI, `better-sqlite3` for storage. No test framework yet (verification is manual on real data per spec).

**Spec:** `docs/superpowers/specs/2026-05-22-mvp-design.md`

### Task 3: Claude Code JSONL parser

**Files:** `src/ingest/claude_code.ts`

- [ ] **Step 1:** Inspect a real JSONL file from the user's `~/.claude/projects/` to confirm field shapes match assumptions. Pick the most recent JSONL file and read the first 50 lines.

```bash
ls -lt ~/.claude/projects/*/ | head
# pick one, then:
head -50 <path-to-recent-jsonl>
```

Look for:
- `sessionId` field shape — should be UUID
- `messageId` field — used for dedup
- `message.usage.input_tokens` / `output_tokens` — primary token source
- `costUSD` field on assistant turns — may or may not be present
- Tool calls — entries with `type: "tool_use"` or `tool_name: "Bash"`, plus their corresponding `tool_result` entries with stdout

**If field names differ from the spec, update the spec FIRST**, then write the parser to match. Don't write a parser against assumed shapes.

- [ ] **Step 2:** Implement `parseJsonlFile(path: string): { sessions: SessionEvent[]; toolCommitHints: ToolCommitHint[] }`.

Logic:
1. Stream the file line by line (`readline` on a `createReadStream`)
2. `JSON.parse` each non-empty line, swallow parse errors with a counter (log at the end)
3. Maintain a `seenMessageIds: Set<string>` and skip duplicates
4. Group entries by `sessionId`
5. For each group:
   - Sort by timestamp
   - Detect logical-session splits: if gap >10 min between consecutive entries, split into a new logical session (append `:1`, `:2` etc. to session_id to keep them unique)
   - Sum `input_tokens` and `output_tokens` across assistant turns
   - Sum `costUSD` if present (else leave at 0 — V0.2 will add a price table)
   - First entry's timestamp = session start; last = session_end_ms
   - Model: take the `model` from the first assistant turn that has one
   - Emit one `SessionEvent` per logical session
   - Skip sessions where `tokens_in + tokens_out == 0` (counter for "skipped" in return)
6. Scan for tool-call entries:
   - For each `tool_use` of `Bash` whose input string starts with `git commit`
   - Find the corresponding `tool_result` (same id or adjacent entry)
   - Regex stdout: `/^\[\S+\s+([a-f0-9]{7,40})\]/m` — capture group 1 is commit hash (may be short — keep both)
   - Emit `ToolCommitHint { session_id, commit_hash, timestamp }`

- [ ] **Step 3:** Implement `findJsonlFilesModifiedSince(sinceMs: number): string[]`.

`fs.readdirSync(claudeProjectsDir(), { withFileTypes: true })` to enumerate project dirs, then for each project dir list `.jsonl` files, stat each, return those with `mtime >= sinceMs`.

- [ ] **Step 4:** Implement `ingestClaudeCode(db, sinceMs): { sessionsIngested: number; toolCommitHints: ToolCommitHint[]; skipped: number; parseErrors: number }`.

Wires the above: list files → parse each → write events to DB → return aggregate counts and the union of all hints (needed for direct attribution in the next module).

Project hash for each session: use the parent directory name of the JSONL file (Claude Code names them after the project path — e.g. `C--Users-bisse-projects-foo`). Hash with SHA256.

- [ ] **Step 5:** Verification — write a one-off script that parses ONE real JSONL file from the author's machine and prints session counts + first hint.


Run on a real file. Eyeball the output. Token counts should be plausible (not all zero). At least one tool commit hint should appear if any session committed.

### Task 4: Git parser

**Files:** `src/ingest/git.ts`

- [ ] **Step 1:** Implement `gitLogSince(cwd: string, sinceMs: number, projectHash: string): CommitEvent[]`.

Run via `child_process.execFileSync`:
```
git -C <cwd> log
  --numstat
  --pretty=format:'COMMIT|%H|%ct|%D'
  --since=@<unix_seconds>
  --no-merges
  HEAD
```

Parse output:
- Lines starting with `COMMIT|` are commit headers — split on `|`
- Following lines until next `COMMIT|` (or EOF) are numstat lines: `<added>\t<removed>\t<path>`
- Aggregate per commit: total added, removed, files_changed (line count)
- `primary_language` = extension of the most-modified file (by added+removed lines); strip leading dot; map common ones (`ts` → TypeScript, etc.) but for V0.1 just store the raw extension
- `branch` = parsed from `%D` (refs) — best-effort, often empty
- Skip commits with zero numstat lines (merges, empty commits)

Wrap the call in try/catch — `cwd` may not be a git repo. Return `[]` if so.

- [ ] **Step 2:** Implement `ingestGit(db, cwd, sinceMs): { commitsIngested: number }`.

Compute project_hash from cwd. Call gitLogSince. Insert events. Return count.

- [ ] **Step 3:** Verification — point it at a real local repo and confirm row counts.


### Task 5: Attribution

**Files:** `src/ingest/attribution.ts`

- [ ] **Step 1:** Implement `attributeDirect(db, hints: ToolCommitHint[]): number`.

For each hint, INSERT OR REPLACE into `attributions` with `tier='direct', confidence=1.0`. Skip hints where the commit_hash doesn't exist in `events` (the commit may not have been ingested yet from a different cwd). Return number of attributions written.

Note: hints may have short hashes; before writing, resolve to full 40-char hash by querying `events WHERE commit_hash LIKE ? || '%'` (limit 1). If multiple matches, skip with a warning.

- [ ] **Step 2:** Implement `attributeInferred(db, windowMinutes: number = 5): number`.

For each commit in `events` of type='commit' that has NO existing attribution:
- Look for sessions where `project_hash` matches AND `session_end_ms` is within `windowMinutes * 60_000` ms of the commit timestamp (before or after, but prefer "before" — session ends, then commit happens)
- If exactly one matches: write attribution with `tier='inferred', confidence=0.5`
- If multiple match: write attribution to the closest-in-time session; tie-break by latest session_end_ms

Return count of attributions written.

- [ ] **Step 3:** Verification — after running attribute step on real data, query the DB:

```sql
SELECT tier, COUNT(*) FROM attributions GROUP BY tier;
```

Direct count should be >0 if the author has used Claude Code's bash tool to commit. Inferred should be >0 for any commits the author made manually after a Claude session.

### Task 6: YPT compute + snapshot writer

**Files:** `src/compute/ypt.ts`

- [ ] **Step 1:** Implement pure `yptScore(inputs): { score: number; provenance: object }`.

```ts
export interface YptInputs {
  direct_attribution_count: number;
  inferred_attribution_count: number;
  total_tokens: number; // tokens_in + tokens_out
}

export function yptScore(i: YptInputs): { score: number; provenance: object } {
  const outcome_signals =
    1.0 * i.direct_attribution_count +
    0.5 * i.inferred_attribution_count;
  const tokens = Math.max(i.total_tokens, 1);
  const token_penalty = 10 * Math.log10(tokens / 10000);
  const score = Number((outcome_signals - token_penalty).toFixed(2));

  return {
    score,
    provenance: {
      formula: 'YPT = Σ(outcome_signals) - 10 · log₁₀(tokens / 10000)',
      inputs: { ...i, outcome_signals, token_penalty },
      academic_source: 'Adapted from OckBench (arxiv 2511.05722)',
      version: 'v0.1',
      notes: 'V0.1: outcomes = attributed commits only. V0.2 will add code survival, self-tags.',
    },
  };
}
```

- [ ] **Step 2:** Implement `computeSnapshotsSince(db, sinceMs: number): number`.

1. Query distinct (date, project_hash) pairs from events where timestamp >= sinceMs
2. For each pair, aggregate:
   - sessions in that UTC day for that project (sum tokens, count)
   - commits in that UTC day for that project (count)
   - attributions joined to those commits, count by tier
3. Compute YPT via `yptScore`
4. Compute cost_per_ship: `attributed_commit_count > 0 ? total_tokens / attributed_commit_count : null`
5. Upsert snapshot row

Use a single SQL aggregation query per snapshot for speed (don't pull all events into Node).

- [ ] **Step 3:** Verification — after sync + compute, dump snapshots:

```bash
sqlite3 ~/.mileage/metrics.db 'SELECT date, project_hash, ypt_score, cost_per_ship_tokens FROM snapshots ORDER BY date DESC LIMIT 10;'
```

Should show recent days with non-null YPT scores in a reasonable range. If YPT is always negative-infinity or always exactly zero, the formula or the inputs are wrong — debug before moving on.

### Task 8: CLI wiring

**Files:** `src/cli.ts`

- [ ] **Step 1:** Wire commander with subcommands.

```ts
import { Command } from 'commander';
import { openDb } from './storage/db';
import { ingestClaudeCode } from './ingest/claude_code';
import { ingestGit } from './ingest/git';
import { attributeDirect, attributeInferred } from './ingest/attribution';
import { computeSnapshotsSince } from './compute/ypt';
import { renderLast7Days, renderExplain } from './render/show';
import { projectHashFromCwd } from './storage/paths';

const program = new Command();
program.name('mileage').version('0.1.0').description('Measure your AI coding efficiency');

program
  .command('sync')
  .option('--since <duration>', 'how far back to sync (e.g. 7d)', '7d')
  .action(async (opts) => {
    const sinceMs = parseSince(opts.since);
    const db = openDb();
    const cc = ingestClaudeCode(db, sinceMs);
    const git = ingestGit(db, process.cwd(), sinceMs);
    const direct = attributeDirect(db, cc.toolCommitHints);
    const inferred = attributeInferred(db);
    const snaps = computeSnapshotsSince(db, sinceMs);
    console.log(`Synced ${cc.sessionsIngested} sessions, ${git.commitsIngested} commits, ${direct + inferred} attributions (${direct} direct, ${inferred} inferred), ${snaps} snapshots`);
    if (cc.parseErrors > 0) console.warn(`  warning: ${cc.parseErrors} JSONL parse errors (skipped)`);
    if (cc.skipped > 0) console.warn(`  warning: ${cc.skipped} sessions skipped (zero tokens)`);
  });

program
  .command('show')
  .option('--project <hash>', 'restrict to one project')
  .option('--days <n>', 'window in days', '7')
  .action((opts) => {
    const db = openDb();
    console.log(renderLast7Days(db, opts.project));
  });

program
  .command('explain <metric>')
  .action((metric) => {
    const db = openDb();
    console.log(renderExplain(db, metric));
  });

program
  .command('debug:list-projects')
  .action(() => {
    const db = openDb();
    const rows = db.prepare('SELECT DISTINCT project_hash, COUNT(*) as n FROM events GROUP BY project_hash ORDER BY n DESC').all();
    console.table(rows);
  });

program.parseAsync();

function parseSince(s: string): number {
  const m = s.match(/^(\d+)([dh])$/);
  if (!m) throw new Error(`bad --since: ${s} (try 7d or 24h)`);
  const n = parseInt(m[1], 10);
  const mult = m[2] === 'd' ? 86_400_000 : 3_600_000;
  return Date.now() - n * mult;
}
```
