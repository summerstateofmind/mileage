# Mileage V0.2 — Cost-Wedge Design Spec

**Date:** 2026-05-22
**Status:** Draft
**Goal:** Make `mileage show` answer "what did my AI spend actually buy me this week?" in dollars, viscerally, in the first 5 seconds of reading. Surface waste sessions. Earn the user's first-value moment.

This spec covers V0.2 in two phases:
- **Phase 1** = the cost wedge itself (dollar surfacing + restructured display + waste detection). Ship-blocking; this is the wedge.
- **Phase 2** = sticky features that drive retention (self-tag, heatmap, post-commit hook, weekly report).

---

## Phase 1 — The Cost Wedge

### User journey

```
[1: Author runs `mileage sync --since 7d` from a repo]
  -> Existing V0.1 ingest runs (JSONL + git + attribution + snapshots)
  -> NEW: per-session USD cost is computed from token usage × model pricing
  -> NEW: snapshots include total_cost_usd, cost_per_ship_usd populated
       |
[2: Author runs `mileage show`]
  -> Output leads with $ spend (this week vs last week)
  -> Cost-per-Ship in dollars is the headline metric
  -> Top 3-5 most expensive sessions are listed with timestamps
  -> Waste sessions (high $ + 0 attributed commits) flagged with ⚠
  -> YPT moves to below-the-fold ("YPT 42 — see `mileage explain ypt`")
       |
[3: The first-value moment]
  -> User sees a number that surprises them ($487? On just 4 PRs?)
  -> They scan the top sessions and recognize the one that wasted $84
  -> They have data they can act on tomorrow (cut Opus for routine work, etc.)
```

### Key decisions

- **Cost-per-Ship is the hero, not YPT.** Per the Path A pivot — dollars are the visceral first-value moment.
- **Pricing table is hardcoded in V0.2.** Single file (`src/pricing/models.ts`), easy to update. V0.3+ can pull from a service or let users override.
- **Cost computed at sync time, stored on the session row.** Querying recomputed costs at render time is slower and harder to debug.
- **Cache-aware token weighting.** Anthropic bills cache_creation at 1.25× input rate and cache_read at 0.1× input rate. We track all three buckets separately and apply the right multipliers.
- **Model unknown → use a fallback rate, but warn.** New Claude or GPT model that isn't in the table yet → log a warning + use the closest-tier known model's rate. Better than failing silently.
- **"Waste session" definition for V0.2:** USD cost > $5 AND zero attributed commits AND no self-tag (V0.2 Phase 2). The $5 threshold is arbitrary; revisit after dogfooding.
- **"Top expensive sessions" surfacing:** top 5 by USD cost in the window, each with `(timestamp, duration_hint, attributed_commit_count, optional_status_marker)`. No code, no prompts — just the cost shape.
- **YPT stays on V0.1.1 formula for V0.2.** Math upgrade is V0.3. Don't conflate UX restructure with formula change.

### Database changes

Add columns to `snapshots` table (carry-forward from V0.1; no migration needed since we already have `total_cost_usd` and `cost_per_ship_usd`).

Add to `events` table (sessions only):
- `effective_tokens_in` INTEGER — the actual billed-equivalent input tokens (input + 1.25·cache_creation + 0.1·cache_read)
- `pricing_version` TEXT — `"2026-05-22"`, lets us recompute historical costs if pricing changes

Backfill: a one-off `mileage recompute-costs` command (hidden subcommand, V0.2-only) that walks all sessions and populates the new columns. Alternative: just blow away the DB and re-sync (we're pre-launch).

**`src/pricing/models.ts`** — pricing table + lookup

  input_per_mtok: number;       // USD per million input tokens

  cache_read_per_mtok: number;   // 0.1  × input by default
  output_per_mtok: number;

export function pricingFor(modelId: string): ModelPricing { ... }

**`src/compute/cost.ts`** — token → USD
```ts
export interface TokenUsage {

```

**Updated `src/ingest/claude_code.ts`** — emit per-session cache-creation and cache-read counts so the cost can be computed from real billing buckets.

**Updated `src/render/show.ts`** — new top-down output structure.

### New display structure

```
Mileage  ·  this week (May 16 – May 22)

  Spend             $487   ▲ +212% vs prior week ($156)
  Outcomes          4 PRs shipped (vs 5 prior week)
  Cost-per-Ship     $122 / PR   (vs $31 prior week — what changed?)

  Where it went (top 5 sessions this week):
    $94  Wed 14:32 · 3h12m · 0 commits  ⚠ waste session
    $61  Thu 09:15 · 2h20m · 1 commit
    $58  Fri 16:00 · 1h45m · 2 commits + tests
    $54  Mon 11:45 · 2h00m · 1 commit (later reverted) ⚠
    $48  Tue 13:30 · 1h30m · 1 commit

  YPT 42 (your trailing 30d median: 39) — see `mileage explain ypt`
```

Key UX choices:
- Spend leads, in dollars, with delta from prior week
- Outcomes shown as "PRs" (when GitHub data present) or "commits" (git-only fallback)
- Cost-per-Ship is the highlighted Big Number
- "Where it went" is the visceral hook — concrete sessions, dollar amounts, optional warning markers
- YPT is one line at the bottom, deliberately understated until the math upgrade in V0.3

### Edge cases (Phase 1)

- **No model in pricing table.** Log warning + use closest known model's rate. Mark session with `pricing_fallback: true` so we can audit later.
- **Session with `tokens=0`.** Already skipped in V0.1 ingest.
- **Session with very high cost but very short duration.** Likely a cache-heavy session (lots of context, little new tokens). Leave alone — it's accurate.
- **No sessions in the window.** Render: "No AI sessions in the last 7 days. Run from a repo where you used Claude Code recently, or extend with `--since 30d`."
- **First week of data, no prior week to compare.** Hide the delta line, show absolute numbers only.
- **Currency.** USD only for V0.2. Multi-currency is a backlog item.

### Out of scope (Phase 1)

- Self-tag system (Phase 2)
- Heatmap (Phase 2)
- Git post-commit hook (Phase 2)
- Weekly report subcommand (Phase 2)
- Cross-tool comparison (V0.4)
- Quality Mode (V0.3)
- LLM-as-judge (V0.3)
- YPT formula upgrade (V0.3)

## Phase 2 — Sticky features

### Self-tag system (`mileage tag`)

Interactive, one-keypress tagging of recent unscored sessions. When the user runs `mileage tag`:

```
$ mileage tag

Session 1 of 3 (Wed 14:32, $94, 3h12m, 0 commits):
  What happened?  [s]hipped  [e]xploring  [d]ebugging  [x] dead-end  [k]ip
> _
```

- Tags become primary outcome signal in YPT (Phase 2 → V0.3 formula upgrade)
- "Exploring" sessions excluded from waste-session detection (research ≠ waste)
- "Dead-end" sessions weighted in waste-ratio computation
- Tag stored in new `session_tags` table with `(session_id, tag, tagged_at)`

### Heatmap (`mileage show --heatmap`)

GitHub-contribution-graph style, 90-day window. Color encoding:
- Color = Cost-per-Ship ratio (green = efficient, yellow = neutral, red = expensive)
- Cell brightness = absolute spend (darker = more spend)
- Empty cell = no AI sessions that day

Terminal renderer using ANSI 256-color or truecolor. Falls back to monochrome blocks if `NO_COLOR` set.

No streaks — research warned they're anxiety-inducing.

### Git post-commit hook (`mileage install-hook`)

`mileage install-hook` writes a `.git/hooks/post-commit` script:
```sh
#!/bin/sh
mileage sync --silent &
```

Plus a sidecar file at `~/.mileage/active-sessions.jsonl` updated by claude_code parser whenever a session is currently in progress (recent JSONL write within 5 min). The hook reads this and writes High-tier attribution `(session_id, commit_hash, tier=high, conf=0.75)`.

`mileage uninstall-hook` removes it cleanly.

### Weekly report (`mileage report --week`)

Copy-pasteable markdown summary for sharing on social/blog. The brand-building artifact.

```
$ mileage report --week

# My Mileage Week — May 16-22, 2026

**Spend:** $487 (+212% vs prior week)
**Cost-per-Ship:** $122/PR (vs $31)
**Top model:** Claude Opus 4.7 (73% of spend)

## Surprising things
- A 3-hour Wednesday debug session cost $94 and shipped nothing. I should
  have switched models 30 minutes in.
- Sonnet sessions averaged $14/commit; Opus sessions averaged $76/commit.
  For routine work that's a 5× premium I'm not sure I'm getting back.

[mileage 0.2 · github.com/mileage-cli/mileage]
```

Output is plain markdown, copy-paste friendly. The author dogfoods this format publicly every week — it becomes the standards-play seed and the personal-brand artifact.

- YPT formula upgrade
- Code Survival Rate computation

## Open questions (decide before V0.2 ship)

1. **Pricing table source of truth.** Hardcoded in `models.ts`, fetched from a community-maintained JSON, or user-overridable via `~/.mileage/pricing.json`? V0.2 ships hardcoded; revisit after dogfooding.
2. **Waste threshold ($5).** Arbitrary. After 2 weeks of dogfooding, look at the distribution and tune. V0.3 could make it a percentile (top 10% cost sessions with no outcome are waste).
3. **Currency.** USD only for V0.2. Backlog: multi-currency with FX from a daily snapshot.
4. **Session "description" inference.** The "Wed 14:32 · 3h12m · 0 commits" line is useful but could be richer. Without reading prompts/code (privacy line), the best signals are: project context (cwd → repo name), time block (morning/afternoon/late), associated commits. Worth doing in V0.2 if cheap.
